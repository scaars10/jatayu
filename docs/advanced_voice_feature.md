# Advanced Voice Interface and Wake Word

## Overview
To mimic the seamless interaction of an assistant like Jarvis, the user must be able to speak to the AI without needing to click buttons or hold a microphone. This feature introduces continuous listening (wake word detection) and low-latency, emotive Text-to-Speech (TTS).

## Key Capabilities
- **Wake Word Activation**: Constantly listening locally for a trigger phrase (e.g., "Hey Jatayu," "Jarvis") without sending audio to the cloud.
- **Conversational Interruption**: The ability for the user to interrupt the AI while it is speaking, instantly stopping the TTS and listening to the new command.
- **Emotive Voice Synthesis**: Using advanced TTS models to generate highly realistic, expressive voices that convey tone rather than robotic readouts.
- **Audio Context**: The ability to "hear" background noise (e.g., recognizing a dog barking or a baby crying) and adapting the response.

## Implementation Plan

### Phase 1: Local Wake Word Detection
1. **Engine Selection**: Integrate a lightweight, on-device wake word engine like **Porcupine (Picovoice)** or **Snowboy**.
2. **Audio Pipeline**: Establish a continuous audio stream from the microphone using PyAudio. When the wake word is detected, trigger the main recording phase.

### Phase 2: Low-Latency TTS Integration
1. **Provider Integration**: Integrate a high-quality TTS provider like ElevenLabs or OpenAI's TTS API.
2. **Streaming Audio**: Implement audio streaming so playback begins as soon as the first sentence is generated by the LLM, drastically reducing perceived latency.

### Phase 3: Full Duplex & Interruption (Barge-in)
1. **Voice Activity Detection (VAD)**: Use WebRTC VAD or Silero VAD to detect when the user starts speaking while the AI is talking.
2. **Buffer Management**: Upon interruption, instantly flush the TTS audio buffer and feed the interrupted context back into the LLM so it knows where it was cut off.

## Technical Stack
- **Wake Word**: Picovoice Porcupine (Python SDK).
- **Audio IO**: `pyaudio` or `sounddevice`.
- **VAD**: `silero-vad`.
- **TTS API**: ElevenLabs / OpenAI / Google Cloud TTS.
